Agent Runtime Control Tower

Control layer for AI agents in production.

A FastAPI service for governing agent execution across MCP servers with approvals, policy checks, incident visibility, and trace-friendly runtime records. Built to show the layer most AI demos skip: who can run what, against which tools, under which constraints.

Goal: make AI agent operations observable, tenant-scoped, and stoppable before bad tool access reaches production systems.
2 Seeded organizations with isolated runtime scope
2 MCP servers with different trust and risk tiers
3 Control paths: auto-approve, human approval, hard block

Core Runtime Decisions

  • Reject cross-tenant requests before execution.
  • Escalate risky runs to approval when budget or tool-call thresholds are exceeded.
  • Block policy-restricted MCP targets and emit incidents automatically.
  • Capture simulated tool traces for every run to support review and replay.

Why This Matters

  • Most agent demos stop at the model call and ignore operational safety.
  • Enterprise teams need approval routing, auditability, and clear policy boundaries.
  • This project focuses on the backend control plane around agents, not just the agent itself.
  • It is designed as a portfolio-ready proof of system design, reliability thinking, and AI governance.
